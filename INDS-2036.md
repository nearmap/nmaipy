# INDS-2036 - add dominant shape/material columns

## Objective

Add dominant material and dominant shape categorical columns to outputs

## Background

Currently, when nmaipy exporter creates attribute columns for a roof that include shape and material attributes, it creates a group of columns per class (e.g., for asphalt shingle, you get _present, _area, _ratio, and _confidence, something like that). 

End users of our data often want to know what the "dominant" shape and material of a roof is. The objective of this ticket is to add columns to all output files that contain data describing roofs: 

* dominant_material (examples: "shingle", "clay_tile", "wood_shake" etc.)
* dominant_material_confidence 
* dominant_material_area_sqft
* dominant_material_ratio
* dominant_shape (examples: "hip", "gable", "flat", "mansard" etc.)
* dominant_shape_confidence 
* dominant_shape_area_sqft
* dominant_shape_ratio

The `dominant_shape` and `dominant_material` columns should be categorical columns whose values will be the class name of the shape or material feature determined to be "dominant" according to algorithms set in this initiative. 

## Algorithm for dominant material 

Choose material class that is flagged as dominant=Y in the per-feature outputs

If multiple classes are flagged as dominant, pick the one with highest ratio. 

## Algorithm for dominant shape

Choose shape class with largest ratio above 50% confidence

## Implementation

* Add an argument that must be flagged as True in Export in order to create these post-processed columns. Default that argument to True so they are output by default. 
* If the argument is true, create these columns whenever the per-class-shape/material attributes are being created. if shape/material not included in export, don't create these columns.
