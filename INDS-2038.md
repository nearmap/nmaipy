# INDS-2038

## Title
Do not populate feature_is_present, feature_area_sqft etc. if no survey resources found - applies across export files (rollup, roof etc.)

## Description
When exports include AI packs (observational data) and roof age (conflation data) the by-feature columns in roof.csv and rollup.csv are being populated with N/0/0/0/0 even when no survey was found from which to pull observational data. 

## Examples
Example output: s3://data-extraction-us-east-1/AIExporterDataExtraction/test/test_2026-03-10_LOCATION_HU_RIJRA_12312025_forDataAugmentation_1

### Case 1: No imagery coverage, roof age available
See rollup.csv, `aoi_id = 4514`: 
* system_version, link and survey_date are empty (indicating no observational data was obtained from Features API)
* duct_present/count/total_area etc. are populated as N/zeros despite not having any imagery to make that determination (no features API result, no Nearmap imagery coverage) (same with all other analogous columns for other feature classes)
* primary_roof_instance_roof_age_xxx columns are populated because we did get a result from the roof age API (a non-imagery-based result)

### Case 2: Imagery coverage exists but was missed by Feature API calls, roof age available 
See rollup.csv, `aoi_id = 4560`:
* system_version, link and survey_date are empty (indicating no observational data was obtained from Features API)
* duct_present/count/total_area etc. are populated as N/zeros despite not having any response from Features API to make that determination 
* primary_roof_instance_roof_age_xxx columns are populated with evidence type 7, indicating we have multiple images and detected a roof replacement
