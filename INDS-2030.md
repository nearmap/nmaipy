# INDS-2030 Postprocess RSI on roofs and building lifecycles to capture RSI when it is provided on building lifecycle instead of roof 

## Background

RSI is the Nearmap roof condition score on a scale of 0-100 and measures the overall condition of a roof based on Gen 6 detections in Nearmap imagery. A 100 is a perfect score and a 0 is a totally damaged roof. 

The "Roof" class in Gen 6 ontology explicitly excludes a hole in the roof, which means if we calculate RSI on a roof that visibly has a hole in it, the calculation will exclude the portion of the roof that has the damage and will result in a high score. 

This is obviously a problem, so the Features API has the following logic in place: 
* If no structural damage is present on / near the roof polygon, calculate RSI on roof polygon
* If structural damage is present on / near the roof polygon, calculate RSI on the building lifecycle polygon instead

(Building Lifecycle is a combination of Roof, Translucent Roofing, Building, Building with Structural Damage, and Building with Structural Loss. This is designed to capture a building in all lifecycle stages from construction to damage.)

If you request both roof and Building Lifecycle classes from the Features API and include=roofSpotlightIndex, the response will assign RSI only to the appropriate class in each pair (of Roof -> Building Lifecycle) based on the structural damage logic. 

`nmaipy` does not currently have a way to provide the "best" RSI for each record that incorporates this logic. This ticket is intended to solve for that. 

### Nomenclature

* `roof` = Gen 6 roof class (excludes structural damage / hole in roof)
* `building lifecycle` = Gen 6 composite class of roof, translucent roofing, building, building with structural damage, and building with structural loss (all constituent gen 6 classes)
* `footprint` = outline of building/roof on which RSI was calculated. This is merely a concept for the purposes of this ticket / the RSI logic, it is not a Gen 6 class. The `footprint` for each building in Nearmap imagery is almost always the roof outline, except in the case of structural damage presence, in which case the `footprint` is the building lifecycle outline (which encompasses the damage/hole). 

## Objective  

Create a column in rollup, building and roof outputs from nmaipy that captures the RSI from the relevant `footprint`.
